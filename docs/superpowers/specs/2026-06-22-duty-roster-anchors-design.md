# 每队独立接龙节点设计

## 背景

现在管理页只有“接龙起点”，它默认表示“当月 1 号前一天排到谁”。这个模型能从月初开始排，但不适合接已有排期。

实际使用里，负责人经常知道某一天的真实值班人，比如 `2026-06-22` 后端是谁，只想让系统从这一天往后接着排。基准日之前的排期不应该被重算。

## 目标

- 每个团队独立维护接龙节点。
- 节点默认表示“基准日当天值班人”。
- 也支持“基准日前一天值班人”，方便只知道前一天是谁的场景。
- 排某一天时，只使用当前团队在这一天之前最近的节点。
- 节点之前已经发布过的排期尽量保持不动。
- 旧数据里的 `last` 继续可读，自动转成节点使用。

## 非目标

- 不做全局节点。前端、后端、测试的节点互不影响。
- 不改飞书提醒的读取方式。提醒仍然从已发布的 `months` 里找当天排班。
- 不引入后端服务或数据库。

## 数据结构

团队配置新增 `anchors` 字段：

```json
{
  "name": "后端",
  "members": ["A", "B", "C", "D"],
  "last": "A",
  "anchors": [
    { "date": "2026-06-01", "mode": "currentDay", "person": "A" },
    { "date": "2026-06-22", "mode": "currentDay", "person": "C" }
  ]
}
```

字段含义：

- `date`：基准日，格式是 `YYYY-MM-DD`。
- `mode`：`currentDay` 表示 `person` 是当天值班人；`previousDay` 表示 `person` 是前一天值班人。
- `person`：成员名单里的姓名。

保留 `last` 是为了兼容旧数据和旧文档语义。新逻辑优先读 `anchors`，没有节点时再用 `last` 推导一个月初节点。

## 排班规则

对每个团队单独计算。

1. 先整理这个团队的节点，过滤无效日期和不在成员名单里的人员。
2. 节点按日期升序排列。
3. 排某一天时，找日期小于等于当天的最近节点。
4. 如果节点是 `currentDay`，节点当天就是 `person`。
5. 如果节点是 `previousDay`，节点当天是 `person` 的下一个人。
6. 节点后的日期按成员名单循环推进。
7. 如果当天之前没有任何节点，就回退到旧的 `last` 逻辑，保持现有行为。

例子：

```text
名单：A, B, C, D
节点：2026-06-22 / currentDay / C

06-22 C
06-23 D
06-24 A
```

```text
名单：A, B, C, D
节点：2026-06-22 / previousDay / C

06-22 D
06-23 A
06-24 B
```

## 管理页交互

每个团队卡片里增加“接龙节点”区域，放在团队名称和成员名单附近。

每行节点包含：

```text
[日期] [当天值班人/前一天值班人] [人员] [删除]
```

新增节点默认：

- 日期：当天。
- 模式：当天值班人。
- 人员：当前团队第一个成员。

节点变更会标记为待发布。发布时生成当前月排班并写入 `data/schedule.json`。

## 发布和历史保护

发布时仍然只写入当前展示月的 `months[YYYY-MM]`，并保留远端已有的其他月份。

如果当前月远端已有排班，生成时可以读取远端结果，节点日期之前的已发布排班保持原样。节点当天和之后按新节点重新生成。这样新增中途节点时，不会把月初已经生效的排班一起改掉。

如果当前月还没有快照，也不能直接从月初整月重算。发布时要先用远端旧规则推导出原计划：未变更的团队整月沿用原计划；只改了名单或顺序但没有明确新节点的团队，当前月也沿用原计划。只有新增或修改了明确节点的团队，才从节点日期开始按新规则顺排；如果节点日期早于发布日，则从发布日开始生效。

## 兼容策略

- 旧配置没有 `anchors` 时，按现有 `last` 生成。
- 打开管理页时，如果团队没有节点，可以显示一个从当前 `last` 推导出的默认节点。
- 写回 `schedule.json` 时保存 `anchors`，同时保留 `last`，让旧逻辑仍有兜底。

## 测试重点

- `currentDay` 节点当天就是填写的人。
- `previousDay` 节点当天是下一个人。
- 多个节点时，后面的节点只影响自己当天及之后。
- 三个团队节点互不影响。
- 旧数据只有 `last` 时行为不变。
- 只改名单或顺序、没有明确新节点时，当前月已发布排班不变。
- 发布合并远端 `months` 时不丢其他月份。
- 飞书提醒脚本仍能读取已发布排班。
