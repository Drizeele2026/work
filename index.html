这个工具怎么工作

它是一个零后端的值班排班页面。公开成员打开页面直接看排班，负责人本地配置好 token 后，再把排班数据写回 GitHub 仓库里的文件。

数据放在哪里

  • 页面文件是 index.html。
  • 排班数据建议放在 data/schedule.json。
  • 介绍内容可以放在 data/about.md，后续再接页面渲染。
  • 仓库地址是 Drizeele2026/work。

为什么这么设计

团队成员不需要登录。负责人只在可信电脑上配置一次 GitHub PAT，保存就是一次普通 commit，天然有历史记录。

当前版本

这版把普通查看和管理页面分开。团队成员看排班和复制内容，负责人使用单独管理地址改成员、重排后续月份和保存。
