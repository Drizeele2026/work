这个工具怎么工作

它是一个零后端的值班排班页面。公开成员打开页面直接看排班，负责人本地配置好 token 后，再把排班数据写回 GitHub 仓库里的文件。

数据放在哪里

  • 页面文件是 index.html。
  • 排班数据建议放在 data/schedule.json。
  • 介绍内容可以放在 data/about.md，后续再接页面渲染。
  • 仓库地址是 Drizeele2026/work。

为什么这么设计

这个工具不需要服务器。公开页直接读取仓库里的排班文件，团队成员打开链接就能看，不需要登录，也不需要配置 token。

需要改排班时，只有负责人进入管理地址，在可信电脑上配置一次 GitHub PAT。点击写入仓库时，本质上就是把新的 data/schedule.json 提交到 GitHub。

所以仓库就是持久化存储，commit 就是保存记录。每次修改都有提交历史，出了问题可以对比、追溯，也可以回滚。
